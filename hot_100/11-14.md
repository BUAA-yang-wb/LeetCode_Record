# LeetCode 热题题解汇总

## 239. 滑动窗口最大值

### 代码

```python
from typing import List
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        q = deque()  # 存下标，保证 nums[q] 单调递减
        ans = []

        for i, x in enumerate(nums):
            # 1. 移除已经滑出窗口的下标
            while q and q[0] <= i - k:
                q.popleft()

            # 2. 维护单调递减队列
            # 如果当前元素 >= 队尾元素，那么队尾不可能再成为最大值
            while q and nums[q[-1]] <= x:
                q.pop()

            # 3. 当前下标入队
            q.append(i)

            # 4. 从第一个完整窗口开始记录答案
            if i >= k - 1:
                ans.append(nums[q[0]])

        return ans
```

### 简要解析

这题的核心数据结构是 **单调队列**。

队列中存的是数组下标，而不是元素值。这样既能通过 `nums[q[0]]` 拿到当前窗口最大值，也能通过下标判断元素是否已经滑出窗口。

队列维护的性质是：

```text
nums[q[0]] >= nums[q[1]] >= nums[q[2]] ...
```

也就是队头对应的元素永远是当前窗口中的最大值。

每次遍历到新元素 `nums[i]` 时，主要做三件事：

1. 删除已经不在窗口范围内的队头元素；
2. 从队尾删除所有小于等于当前元素的下标；
3. 把当前下标加入队尾。

为什么可以删除队尾中比当前元素小的元素？因为当前元素比它们更大，而且位置更靠右，会更晚离开窗口，所以它们之后不可能再成为最大值。

这题的重点是理解：

> 单调队列只保留窗口中“未来仍有机会成为最大值”的元素。

## 76. 最小覆盖子串

### 代码

```python
from collections import Counter, defaultdict

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        need = Counter(t)          # t 中每个字符需要的次数
        window = defaultdict(int)  # 当前窗口中每个字符出现的次数

        left = 0
        valid = 0                  # 已经满足 need 要求的字符种类数

        min_len = float("inf")
        start = 0

        for right, ch in enumerate(s):
            # 1. 右边界扩张窗口
            if ch in need:
                window[ch] += 1
                if window[ch] == need[ch]:
                    valid += 1

            # 2. 当窗口已经覆盖 t，尝试收缩左边界
            while valid == len(need):
                if right - left + 1 < min_len:
                    min_len = right - left + 1
                    start = left

                left_ch = s[left]

                if left_ch in need:
                    if window[left_ch] == need[left_ch]:
                        valid -= 1
                    window[left_ch] -= 1

                left += 1

        if min_len == float("inf"):
            return ""

        return s[start:start + min_len]
```

### 简要解析

这题的核心是 **滑动窗口 + 哈希表计数**。

`need` 记录目标字符串 `t` 中每个字符需要出现多少次，`window` 记录当前窗口中每个字符出现了多少次。

`valid` 表示当前窗口中已经满足需求的字符种类数。注意，它不是字符总数，而是字符种类数。

当：

```python
valid == len(need)
```

说明当前窗口已经包含了 `t` 中所需的全部字符，此时就可以尝试移动左边界，让窗口尽可能变短。

缩小窗口时，最关键的是这段：

```python
if window[left_ch] == need[left_ch]:
    valid -= 1
window[left_ch] -= 1
```

它的含义是：如果左边界字符在当前窗口中刚好满足需求，那么一旦把它移出窗口，这类字符就不再满足要求了，所以要先让 `valid -= 1`，再减少窗口计数。

这题的模板可以记成：

```python
for right in range(len(s)):
    扩大窗口

    while 窗口满足条件:
        更新答案
        缩小窗口
```

这类题一般用于解决：

> 最短、最长、满足某种条件的连续子串问题。

## 53. 最大子数组和

### 代码

```python
from typing import List

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        cur = nums[0]   # 以当前位置结尾的最大子数组和
        ans = nums[0]   # 全局最大子数组和

        for i in range(1, len(nums)):
            cur = max(nums[i], cur + nums[i])
            ans = max(ans, cur)

        return ans
```

### 简要解析

这题的经典做法是 **Kadane 算法**，本质是 **动态规划的空间压缩**。

定义：

```text
cur 表示以当前元素结尾的最大连续子数组和
ans 表示遍历过程中出现过的最大连续子数组和
```

对于当前位置 `i`，只有两个选择：

1. 接上前面的子数组；
2. 不接前面的，从当前元素重新开始。

所以状态转移是：

```python
cur = max(nums[i], cur + nums[i])
```

这题可以记住一句话：

> 如果前面的和对当前元素有帮助，就接上；如果是拖累，就从当前元素重新开始。

## 56. 合并区间

### 代码

```python
from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key=lambda x: x[0])

        ans = []

        for start, end in intervals:
            # ans 为空，或者当前区间和上一个区间没有重叠
            if not ans or start > ans[-1][1]:
                ans.append([start, end])
            else:
                # 有重叠，更新上一个区间的右端点
                ans[-1][1] = max(ans[-1][1], end)

        return ans
```

### 简要解析

这题的核心是 **排序 + 贪心合并**。

先按照区间左端点排序：

```python
intervals.sort(key=lambda x: x[0])
```

排序之后，区间左端点从小到大排列。这样每次只需要比较当前区间和答案数组中的最后一个区间。

这题可以记住一句话：

> 排序后，只看当前区间能不能接到答案数组最后一个区间后面。
